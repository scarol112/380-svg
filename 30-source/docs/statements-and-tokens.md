# DSL Statements and Token Support

Generated by reading `parser.py`, `ast.py`, and `lexer.py` directly.

---

## Token types

| Token | Meaning |
|---|---|
| `"text"` | Quoted string (kind=`QUOTED`) |
| `<coords>` | COORD pair `x,y` used as begin/end override |
| `<n>px` | Stroke weight (kind=`PX`) |
| `@name` | Element tag (kind=`WORD`, value starts with `@`) |
| `2nd dim` | Second positional measurement — role varies: `width` (rect/textbox), `thickness` (wall), `depth` (window) |
| `A=<h>,<v>` | Absolute canvas-origin offset (kind=`ABSOLUTE`) |
| `align` | Alignment word: `left`, `center`, or `right` |
| `C=<color>` | Per-element color override (kind=`COLOR_ELEM`); also the required argument of the `color` directive (see note 1) |
| `ccw` | Counter-clockwise flag for `arc` |
| `dash style` | Any of: `dashed`, `shortdash`, `dotted`, `center`, `hidden` |
| `degrees` | Single numeric argument to `direction` |
| `dest_x dest_y` | Two leading positional measurements for `moveto` / `lineto` |
| `dim=on/off` | Per-element dimension-display override |
| `element ref` | Bare WORD naming a previously-tagged element (used by `textline`, `textbreak`, `textappend`) |
| `font=<family>` | Font-family override word (kind=`WORD`, prefix `font=`) |
| `length` | First positional measurement (feet/inches) |
| `normal` | Reset `textstyle` to defaults |
| `on/off` | Toggle argument for directives |
| `radius` | First positional measurement for `arc` / `circle` |
| `size` | Bare NUMBER used as font size |
| `start=<angle>` | Arc start angle (compass degrees) |
| `sweep` | Second positional measurement for `arc` (degrees of arc) |
| `swing` | Door swing direction: `left`, `right`, `in`, or `out` |
| `wrap` | Text-wrap flag for `textbox` |

## Notes

**Note 1 — `color/col` directive argument.**
The `color` directive takes a plain color name (WORD token) or quoted CSS string (QUOTED token), not a `C=…` prefixed token. It is mapped to the `C=<color>` column because it serves the same purpose (setting the active drawing color), but the syntax is `color red` or `color "rgb(…)"`, not `C=red`. The quoted CSS form (e.g. `color "rgb(100,200,50)"`) is explicitly handled via `tok.kind == "QUOTED"`.

**Note 2 — QUOTED label on non-text elements.**
`_extract_common` extracts a QUOTED token into a `label` field, but `LineElem`, `WallElem`, `DoorElem`, `WindowElem`, `ArcElem`, `ArrowElem`, and `CircleElem` have no `label` field in their AST nodes. The extracted value is silently discarded without error or warning. Marked as not handled because the parser does not use it. Only `rect` and `textbox` actually store and render the label.

**Note 3 — `point/p` and `_extract_common` discarding.**
`_parse_point` calls `_extract_common`, which internally processes dash styles, QUOTED labels, COORD pairs, and `dim=on/off` — but `PointElem` has no fields for `dash`, `label`, `begin`, `end`, or `show_dims_override`. Those values are silently discarded. Writing `p dashed` or `p 1,2` produces no error and no effect.

**Note 4 — `label/lb` and `dim=on/off`.**
`_parse_label` does not call `_extract_common` and does not check for `dim=on/off`. `LabelElem` has no `show_dims_override` field.

**Note 5 — `lineto/lto` and COORD pairs.**
`_extract_common` is called on `tokens[2:]`, so COORD pairs present after the two positional measurements would be extracted into `begin`/`end` — but `LineToElem` has no `begin`/`end` fields. The values are silently discarded.

**Note 6 — `include/inc` and filenames.**
The filename argument is read as a QUOTED token or a bare WORD. The QUOTED form is accepted but not required (a bare word suffices). A path with spaces must be quoted, since the tokenizer splits on whitespace.

**Note 7 — `direction/dir` raises `ValueError` for measurement tokens.**
`_parse_direction` calls `float(tokens[0].value)` directly. If the argument is a MEASUREMENT token (e.g. `dir 90'`), Python raises a bare `ValueError` instead of a clean `ParseError`. All other statements use `parse_measurement()` which handles feet/inches syntax. `direction` effectively requires a plain NUMBER. (See issue #39.)

**Note 8 — `textbox/tbox` font-size extraction.**
The parser counts all MEASUREMENT/NUMBER tokens and captures a NUMBER as font size only when `dim_count > 2` — i.e., the third or later such token. This skips the two required dimension arguments. The behavior is correct whether dimensions are written as MEASUREMENT tokens (`3' 2'`) or plain NUMBERs (`3 2`), but the logic is subtle and differs from other text elements which simply take the first bare NUMBER as font size. (See issue #40.)

**Note 9 — `textstyle/tstyle` with no arguments is a silent no-op.**
`textstyle` accepts zero arguments and produces a `TextStyleDirective` with both `size` and `font_family` set to `None`. The directive is applied without error but changes nothing.
